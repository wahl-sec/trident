Trident Library Data Storage Module¶

Trident: Data Storage

Trident daemon handling the data collected by all runners under a Trident daemon. @author: Jacob Wahlman

class trident.lib.daemon.data_storage.TridentDataDaemon(daemon_config: trident.lib.daemon.data_storage.TridentDataDaemonConfig)[source]¶

Bases: object

Daemon handling all the reading/writing to the stores used by each TridentRunner. All access goes through this daemon in order to verify that no conflicts occur when writing to the store and that all file handlers are properly opened/closed.

Parameters

daemon_config (TridentDataDaemonConfig) – The config for this data daemon.

merge_store_data() → NoReturn[source]¶

Merges the store data with the existing store data available in the written store. This is used when the store has results written to it from previous iterations.

Raises

JSONDecodeError – Raises JSONDecodeError if the JSON data is not parseable.

store_runner_result(result: Dict) → NoReturn[source]¶

Store the results given in the initialized store. This updates the store in the program and does not write the store to the disk.

Parameters

result (dict) – The result to update the store with in the form of a dictionary.

write_to_store() → NoReturn[source]¶

Writes the current initialized store to the disk on the system for the given store path.

Trident Library Trident Module¶

Trident: Runner Daemon

Trident daemon that controls all runners and couples itself to it’s data storage. The runners are run in a asynchronous environment controlled by the daemon to allow them to run concurrently. @author: Jacob Wahlman

class trident.lib.daemon.trident.TridentDaemon(daemon_config: trident.lib.daemon.trident.TridentDaemonConfig)[source]¶

Bases: object

Main daemon controlling each TridentRunner for the program. Starts execution of the runners, waits for completion and handles any interrupts to the runners.

Parameters

daemon_config (TridentDaemonConfig) – The configuration class for the TridentDaemon.

start_all_runners() → NoReturn[source]¶

Creates a concurrent.futures.Future for each TridentRunner and start each runner asynchronously.

stop_all_runners() → NoReturn[source]¶

Stop execution for all TridentRunner, if it has already started it’s execution then it can’t be halted so a threading.Event will be set to signal for the plugin to halt when seen during execution.

wait_for_runners() → NoReturn[source]¶

Wait for each runner future to report as completed meaning that each TridentRunner has finished. Raises exception for each future that encountered an exception while running.

Raises

Exception – The exception raised while the plugin ran in the TridentRunner.
