Trident Library Runner Module¶

Trident: Runner

Runs and controls the execution of a valid plugin. @author: Jacob Wahlman

class trident.lib.runner.trident.TridentRunner(runner_config: trident.lib.runner.trident.TridentRunnerConfig, runner_id: str, data_daemon_config: trident.lib.daemon.data_storage.TridentDataDaemonConfig)[source]¶

Bases: object

The TridentRunner used to control the execution of each plugin. Starts initialized plugins, evaluates results and calls for storage by the TridentDataDaemon attached to this runner.

Parameters

  • runner_config (TridentRunnerConfig) – The configuration used to define the behavior of this runner.

  • runner_id (str) – The unique identifier for this runner.

  • data_daemon_config (TridentDataDaemonConfig) – The configuration for this runners data daemon.

start_runner() → NoReturn[source]¶

Start the initialized TridentRunner by calling the plugin method execute_plugin with the provided arguments.

Raises

Exception – Re-raised exceptions that occurs in the plugin.

class trident.lib.runner.trident.TridentRunnerConfig(plugin_path: str, plugin_args: Dict[str, Any], resource_queues: Dict[str, List[str]], dont_store_on_error: bool)[source]¶

Bases: object

The configuration for the TridentRunner. Defines the attributes that the runner runs, like plugin instance, module and other info required by the runner.

Parameters

  • plugin_path (str) – The path to the plugin used.

  • plugin_name (str) – The name of the plugin used.

  • plugin_args (dict) – The arguments that should be passed to the plugin when executing.

  • plugin_module (object) – The module object given by import_module from the importlib library.

  • plugin_instance (object) – An instance of the plugin class found in the module.

  • resource_queues (dict) – Queue used to register access to a resource shared between runners to avoid conflicts.

  • dont_store_on_error (bool) – If the runner should store the results accumulated before exiting if an error occured.

  • thread_event (threading.Event) – The event flag used to signal to the plugin that it should exit.
